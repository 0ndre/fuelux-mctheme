Search Box search.js

Combines an input and a button to fire the searched event.

Examples

Wrap the input and search button within .fuelux .search

Sample Methods

Usage

This function includes a simple input and button group that styles with a search icon and fires the searched event.

Via data-attributes

To enable the search control without writing Javascript, add data-initialize="search" to the .search element that you wish to initialize. Such elements that exist when $.ready() executes will be initialized. Any search markup that is programatically created with data-initialize="search" sometime after the initial page load will not immediately initialize. Rather, it will initialize when the mousedown event is fired on it.

Via JavaScript:

Call the search control via JavaScript:

{% highlight js %}$('#mySearch').search();{% endhighlight %}

Methods

$().search();

Attaches search to an input and a button group

.search('destroy')

Removes all functionality, jQuery data, and the markup from the DOM. Returns string of control markup.

{% highlight js %}$('#myScheduler').search('destroy');{% endhighlight %}

.search('disable')

Disables the component

{% highlight js %}$('#myScheduler').search('disable');{% endhighlight %}

.search('enable')

Enables the component

{% highlight js %}$('#myScheduler').search('enable');{% endhighlight %}

Events

Event Type Description
searched.fu.search This event fires when a user presses Enter within the input or clicks the button.
cleared.fu.search This event fires when the user clears the input.

All search events are fired on the .search classed element.

{% highlight js %} $('#mySearch').on('searched.fu.search', function () { // load search results }); {% endhighlight %}
